    **Create `frontend/src/hooks/useStream.ts`:**
    - Custom hook: `useStream()` returning `{ sendMessage, isStreaming, abort }`
    - `sendMessage(sessionId, query, fileList, accessToken)`:
      - Creates new AbortController per request
      - POST fetch to `${MODAL_URL}/chat` with Authorization header, JSON body
      - Reads response.body via ReadableStream + TextDecoder
      - Buffers incomplete SSE lines (split on `\n`, keep incomplete last line)
      - Parses `data: {json}` lines, skips non-data lines
      - Handles event types:
        - `token` → calls `onToken(content)` callback
        - `citations` → calls `onCitations(citations[])` callback
        - `no_results` → calls `onNoResults()` callback
        - `error` → calls `onError(message)` callback
      - `[DONE]` → calls `onDone()` callback
    - `abort()` — calls abortController.abort(), resets isStreaming
    - Callbacks passed as options object to useStream()
    - CRITICAL: Buffer incomplete lines across reads per Research pitfall #1

    **Create `frontend/src/hooks/useAutoResize.ts`:**
    - Hook that takes a textarea ref
    - On input change: reset height to 'auto', then set to scrollHeight + 'px'
    - Max height cap at 200px, then overflow-y: auto

  <name>Task 2: Chat component tree — messages, citations, input</name>
  <files>frontend/src/components/chat/ChatView.tsx, frontend/src/components/chat/MessageList.tsx, frontend/src/components/chat/ChatMessage.tsx, frontend/src/components/chat/CitationBadge.tsx, frontend/src/components/chat/CitationPopover.tsx, frontend/src/components/chat/CitationFooter.tsx, frontend/src/components/chat/ChatInput.tsx, frontend/src/components/chat/StreamingCursor.tsx, frontend/src/components/chat/NoResultsMessage.tsx</files>
  <action>
    Build all chat UI components following locked decisions from CONTEXT.md.

    **`ChatView.tsx`** — Main container:
    - Props: sessionId, indexedSources (for empty state suggestions)
    - State: messages array, activeCitation (for popover), streamingContent
    - Uses useStream hook with callbacks that update message state
    - Full-width layout, flex column, MessageList fills available space, ChatInput pinned bottom
    - Passes abort function to ChatInput for stop generating button

    **`MessageList.tsx`** — Scrollable message area:
    - Renders array of ChatMessage components
    - Auto-scrolls to bottom on new message/streaming content
    - Uses ref + scrollIntoView({ behavior: 'smooth' })

    **`ChatMessage.tsx`** — Single message:
    - User messages: right-aligned or left-aligned with different background (Claude's discretion: user = slightly darker bg, assistant = white/lighter)
    - Assistant messages: render content through citation-aware markdown rendering
    - Split content on `[N]` regex pattern `/(\[\d+\])/g`
    - Non-citation parts rendered via react-markdown
    - Citation `[N]` parts rendered as CitationBadge
    - CRITICAL per Research: Only render citation badges when message is NOT currently streaming (to avoid flickering from partial `[` characters). During streaming, render raw text.
    - After streaming completes, re-render with citation badges
    - Below assistant message: CitationFooter showing compact source summary

    **`CitationBadge.tsx`** — Inline clickable badge:
    - Renders as small blue pill/badge with number (e.g., "1")
    - Hover underline, cursor pointer
    - onClick: passes citation index + anchor element to parent for popover positioning

    **`CitationPopover.tsx`** — Radix Popover:
    - Uses shadcn Popover component (Radix-based)
    - Shows: file name, page/row/slide info (via formatCitationLabel), passage text
    - Passage text truncated to ~200 chars with "Show more" toggle to expand full chunk_text
    - Dismisses on click-outside (Radix built-in)

    **`CitationFooter.tsx`** — Compact source summary:
    - Below each assistant message
    - "Sources: report.pdf p.7, data.csv row 12" — comma-separated list
    - Uses formatCitationLabel for each citation
    - Muted text color, smaller font

    **`ChatInput.tsx`** — Expandable textarea:
    - Uses shadcn Textarea component + useAutoResize hook
    - Starts single-line height, grows as user types
    - Enter sends message (calls onSend prop), Shift+Enter inserts newline
    - Send button (arrow icon) on right side
    - When isStreaming=true: input disabled, send button replaced with "Stop generating" button (square icon). Stop button calls abort().
    - onSend checks for empty input, trims whitespace

    **`StreamingCursor.tsx`** — Blinking cursor:
    - Renders a blinking `|` character via CSS animation
    - Only visible when isStreaming is true
    - Appended after the last streamed token in ChatMessage

    **`NoResultsMessage.tsx`** — No results display:
    - Renders in muted/italic style
    - Message: "I couldn't find relevant information in the provided files. Try rephrasing your question or check if the right files were indexed."
    - Used when useStream receives no_results event
