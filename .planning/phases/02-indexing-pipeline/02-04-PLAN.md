data: {"total_chunks": int}

data: {"embedded": int, "total": int}

event: complete
data: {"files_indexed": int, "total_chunks": int, "skipped_files": [{"file_id": str, "file_name": str, "reason": str}]}

data: {"message": str, "code"?: "empty_folder"|"no_supported_files"}

<task type="auto">
  <name>Task 2: Indexing modal and progress components</name>
  <files>frontend/src/components/indexing/IndexingModal.tsx, frontend/src/components/indexing/FileList.tsx, frontend/src/components/indexing/EmbeddingProgress.tsx, frontend/src/components/indexing/IndexingModal.test.tsx</files>
  <action>
    **IndexingModal.tsx:** shadcn Dialog overlay that orchestrates the indexing flow. Props: open: boolean, driveUrl: string, sessionId: string, token: string, onComplete(result: {filesIndexed, totalChunks, indexedSources}), onCancel(), onError(message: string). Internal state machine:
    - State: "extracting" | "embedding" | "success" | "error"
    - On mount (when open + driveUrl): call streamIndex, parse SSE events via parseSSE
    - During "extracting": render FileList with file statuses updated from SSE extraction events
    - Handle `event: warning` -- show warning text next to the file in FileList (e.g., "Large file - processing may be slow")
    - During "embedding": render EmbeddingProgress with counts from embedding_progress events
    - On "complete" event: show success state "Indexed N files (M chunks)" for 1.5s, then call onComplete and close
    - On "error" event: show error message. If code is "empty_folder" show "This folder is empty". If code is "no_supported_files" show "No supported file types found" with list of what was skipped. Offer close button.
    - Cancel button: abort fetch (AbortController), call onCancel
    - Per CONTEXT.md: cancellation discards all partial data (clean slate) -- onCancel handler in parent should NOT save anything
    - Zero successful files: call onError with appropriate message

    **FileList.tsx:** Renders list of files with status badges. Props: files: Array<{file_id, file_name, status, chunk_count?, reason?, error?, warning?}>. Status badges: "extracting" (blue spinner), "done" (green check), "skipped" (gray), "failed" (red x). Use lucide-react icons. Unsupported/skipped files collapsed at bottom: "N files skipped (unsupported types)" with expandable detail per CONTEXT.md.

    **EmbeddingProgress.tsx:** Single shadcn Progress bar. Props: embedded: number, total: number. Renders "Embedding chunks... 200/312 chunks" with progress bar at embedded/total percentage.

    **IndexingModal.test.tsx:** Test that modal renders in extracting state, shows file list, transitions to embedding state. Mock fetch to return a ReadableStream with encoded SSE events. Test cancel triggers AbortController.abort(). Test error event with "empty_folder" code shows correct message. Test error event with "no_supported_files" code shows correct message.
