  <name>Task 1: POST /index SSE streaming endpoint</name>
  <files>backend/app.py, backend/tests/test_index_endpoint.py, backend/requirements.txt</files>
  <action>
    **Update backend/requirements.txt:** Ensure `fastapi>=0.135.0` (for native EventSourceResponse), `pymupdf`, and `aioresponses` are present.

    **Update backend/app.py:** Add POST /index endpoint using FastAPI's EventSourceResponse (from fastapi.responses import EventSourceResponse; from starlette.sse import ServerSentEvent). The endpoint:

    1. Request body: Pydantic model IndexRequest with drive_url (str) and session_id (str). Auth via Authorization header -- extract token, call get_google_user_id.

    2. Validate drive_url: call extract_drive_id. If None, yield error SSE event and return.

    3. Resolve link: call resolve_drive_link. If folder mimeType, call list_folder_files. If single file, wrap in list.

    4. **Empty folder / no supported files detection (per user decision):**
       - If list_folder_files returns empty list: yield `event: error`, data: `{"message": "This folder is empty", "code": "empty_folder"}` and return.
       - After classifying all files, if zero are supported: yield `event: error`, data: `{"message": "No supported file types found. Supported: Google Docs, Sheets, Slides, PDF, TXT, MD", "code": "no_supported_files"}` and return. Include the skipped_files list in the error data so the frontend knows what was found but unsupported.

    5. **Soft file size limit (per user decision):**
       - Define SIZE_WARNING_BYTES = 50 * 1024 * 1024 (50MB).
       - Before extracting each file, check file["size"] (from Drive metadata). If size > SIZE_WARNING_BYTES, yield `event: warning`, data: `{"file_id": ..., "file_name": ..., "message": "Large file (~{size_mb}MB) - processing may be slow"}`.
       - Still process the file normally after the warning.

    6. Phase 1 -- Extraction: For each file, classify as supported/unsupported. Yield extraction SSE events:
       - `event: extraction`, data: `{file_id, file_name, status: "extracting"}`
       - Export file content, chunk by type (use mimeType to pick chunk_pdf/chunk_sheet/chunk_slides/chunk_text)
       - `event: extraction`, data: `{file_id, file_name, status: "done", chunk_count: N}` or `{..., status: "skipped", reason: "..."}` or `{..., status: "failed", error: "..."}`
       - Continue on per-file errors (best-effort per CONTEXT.md)

    7. Phase 2 -- Embedding: Collect all chunks. Yield `event: embedding_start`, data: `{total_chunks}`. Call embed_chunks with on_progress callback that yields `event: embedding_progress`, data: `{embedded, total}`. Handle embedding failures: retry is inside embed_chunks; if it raises after retries, save what was embedded so far.

    8. Storage: Call save_session with embeddings, chunks, volume. Volume is the module-level `volume` from app.py.

    9. Yield `event: complete`, data: `{files_indexed, total_chunks, skipped_files: [{file_id, file_name, reason}]}`.

    10. Wrap entire stream in try/except: on unhandled error yield `event: error`, data: `{message}`.

    Set timeout=600 on the Modal function decorator (update existing @app.function). Add `pymupdf` to the pip_install list in the Modal image.

    **Write backend/tests/test_index_endpoint.py:** Integration test using httpx AsyncClient with ASGITransport. Mock drive.py functions (resolve_drive_link, list_folder_files, export_file) and auth.py (get_google_user_id) and embedding.py (embed_chunks) and storage.py (save_session). Mock the volume. Test:
    - Valid folder URL: receives extraction events for each file, embedding events, complete event
    - Invalid drive URL: receives error event
    - Single file URL: receives extraction for one file, embedding, complete
    - Unsupported file in folder: receives skipped status for that file, rest processed
    - Empty folder: receives error event with code "empty_folder"
    - Folder with only unsupported files: receives error event with code "no_supported_files"
    - Large file (>50MB): receives warning event before extraction event for that file
    - Event format: each SSE event has correct event name and JSON data shape
