    - grep_live(file_id, keywords, access_token) returns list of up to 15 match dicts with text, matched_keyword, sentence_index, file_id
    - grep_live uses cached text within 5min TTL; re-fetches after expiry
    - grep_live context windows include 1 sentence before and after match
  </behavior>
  <action>
    **Create `backend/staleness.py`:**

    1. Module-level caches:
       - `_staleness_cache: dict[str, tuple[bool, str|None, float]] = {}` (is_stale, error, checked_at)
       - `STALENESS_TTL = 60`
       - Import `_grep_text_cache` from grep module for invalidation (or accept file_id param in invalidate)

    2. `async def get_file_metadata(session, file_id, access_token)` -- GET `https://www.googleapis.com/drive/v3/files/{file_id}?fields=id,name,modifiedTime` with Authorization header. Return `{"file_id": file_id, "error": "not_found"}` on 404, `{"file_id": file_id, "error": "access_denied"}` on 403, else parsed JSON.

    3. `async def check_staleness(file_list, access_token)` -- Check cache first (skip files with valid TTL). For uncached files, `asyncio.gather` all `get_file_metadata` calls via shared `aiohttp.ClientSession`. Compare `modifiedTime > indexed_at` (string comparison works since both are ISO 8601 with Z suffix). Store results in cache. Return `(stale_ids: set, file_errors: dict)`.

    4. `def invalidate_caches(file_id)` -- Pop file_id from `_staleness_cache` and from `grep._grep_text_cache`. Import grep module at function level to avoid circular imports.

    **Create `backend/grep.py`:**

    1. Module-level cache:
       - `_grep_text_cache: dict[str, tuple[str, float]] = {}` (text, fetched_at)
       - `GREP_TEXT_TTL = 300`

    2. `async def extract_keywords(query, model_key="deepseek")` -- LLM prompt per spec: "Extract search keywords... Respond with ONLY a JSON array of strings." Use `get_client(model_key)` from Phase 1 model config. Strip markdown code fences (```` ```json ... ``` ````). `json.loads` the result. Validate it's a list of strings. On parse failure, fallback: split query on spaces, strip stopwords set, filter len > 1.

    3. `async def fetch_and_extract(file_id, access_token)` -- Fetch file content from Drive (reuse Phase 2's file extraction if available, or direct Drive export). Return extracted text string.

    4. `async def grep_live(file_id, keywords, access_token)` -- Check `_grep_text_cache` for valid entry. If expired or missing, call `fetch_and_extract` and cache result. Split text into sentences with `re.split(r'(?<=[.!?])\s+', text)`. Build regex pattern: `"|".join(re.escape(k) for k in keywords)`. Search each sentence with `re.IGNORECASE`. For matches, build context window: sentences[max(0, i-1):min(len, i+2)]. Return list of dicts: `{text, matched_keyword, sentence_index, file_id}`. Cap at 15 results.

    **Modify `backend/chat.py`** (Phase 3's /chat endpoint):

    1. Add imports: `from backend.staleness import check_staleness` and `from backend.grep import extract_keywords, grep_live`.

    2. Modify `event_stream()` inside the /chat endpoint:
       - After loading chunks/embeddings from Volume, run in parallel: `asyncio.gather(check_staleness(body.file_list, access_token), embed_query(body.query))`
       - If `stale_ids` is non-empty, emit SSE event BEFORE any tokens: `data: {"type": "staleness", "files": [{"file_name": f["file_name"], "file_id": f["file_id"], "error": file_errors.get(f["file_id"])}]}` for each file in file_list where file_id in stale_ids
       - Partition chunks: `fresh_mask = [i for i, c in enumerate(chunks) if c["file_id"] not in stale_ids]`
       - Fresh path: if fresh_mask non-empty, run cosine similarity on `chunks[fresh_mask]` / `embeddings[fresh_mask]` using existing `retrieve_mixed()`
       - Stale path: if stale_ids non-empty, call `extract_keywords(query)` THEN `asyncio.gather(*[grep_live(fid, keywords, access_token) for fid in stale_ids])`. Flatten results.
       - Merge: combine retrieved chunks + grep results into the prompt context. Format grep results similarly to chunks for the LLM (text field, file_id, source="grep").
       - Stream LLM response as before.
       - Citations event: include both embedding-sourced and grep-sourced citations. Grep citations get `source: "grep"` marker.

    3. Add `file_list` to the /chat request body model if not already present. Each item: `{file_id, file_name, indexed_at}`.

    4. Guard: if fresh_mask is empty AND no grep results, emit no_results event.
