<task type="auto">
  <name>Task 2: Stabilize IndexingModal onComplete callback to prevent double invocation</name>
  <files>frontend/src/components/indexing/IndexingModal.tsx</files>
  <action>
The root cause: `onComplete` is in the useEffect dependency array (line 209). Since `handleIndexComplete` in AppShell is recreated on `selectedSessionId` change, the useEffect re-fires and re-starts the SSE stream, causing double invocation.

Fix: Use a ref to hold the latest `onComplete` and `onError` callbacks, then remove them from useEffect deps.

1. Add refs at the top of the component:
   ```
   const onCompleteRef = useRef(onComplete);
   const onErrorRef = useRef(onError);
   ```
2. Add sync effects:
   ```
   useEffect(() => { onCompleteRef.current = onComplete; }, [onComplete]);
   useEffect(() => { onErrorRef.current = onError; }, [onError]);
   ```
3. Inside the main useEffect, replace all `onComplete(...)` calls with `onCompleteRef.current(...)` and `onError(...)` with `onErrorRef.current(...)`.
4. Remove `onComplete` and `onError` from the useEffect dependency array. Final deps: `[open, driveUrl, sessionId, token, reset]`.
