<task type="auto">
  <name>Task 2: Refactor AppShell to multi-session state management</name>
  <files>frontend/src/components/app-shell/AppShell.tsx</files>
  <action>
Refactor AppShell.tsx from single-session to multi-session. Per CONTEXT.md locked decisions:

1. **State changes:**
   - Replace `const [sessionId] = useState(() => crypto.randomUUID())` with `const [selectedSessionId, setSelectedSessionId] = useState<string | null>(null)`
   - Replace `const [indexedFiles, setIndexedFiles] = useState<IndexedFile[]>([])` with `const [indexedFilesMap, setIndexedFilesMap] = useState<Map<string, IndexedFile[]>>(new Map())`
   - Add `const [totalChunksMap, setTotalChunksMap] = useState<Map<string, number>>(new Map())`
   - Add `const [deleteTarget, setDeleteTarget] = useState<Chat | null>(null)` for delete dialog
   - Keep `abortRef` via useRef to track useStream abort function

2. **Session switching (onSelect):**
   - Abort any active stream (call abort from useStream if streaming)
   - `setSelectedSessionId(newId)` -- ChatView will remount/reload via key or useEffect

3. **New chat (onCreate):**
   - Generate UUID: `const id = crypto.randomUUID()`
   - Create IndexedDB record: `saveChat({ session_id: id, title: "New Chat", created_at: new Date().toISOString(), last_message_at: new Date().toISOString(), indexed_sources: [] })`
   - Update local chats state, setSelectedSessionId(id)

4. **Rename (onRename):**
   - Call `updateChatTitle(sessionId, newTitle)` from db.ts
   - Update local chats state to reflect new title

5. **Delete (onDelete):**
   - Show DeleteConfirmDialog. On confirm: abort stream if active on that session, call `deleteChat(sessionId)` + `deleteMessages(sessionId)`, remove from local chats state, clear indexedFilesMap entry
   - If deleted session was selected, set selectedSessionId to first remaining chat or null

6. **handleIndexComplete update:**
   - Set title to first indexed source name: `updateChatTitle(selectedSessionId, result.indexedSources[0]?.file_name || "Untitled")`
   - Update indexedFilesMap and totalChunksMap for selectedSessionId
   - Update local chat's indexed_sources: `saveChat({...chat, indexed_sources: [...chat.indexed_sources, ...result.indexedSources.map(f => f.file_id)]})`
   - Refresh local chats state

7. **Render:**
   - Replace inline sidebar with `<Sidebar>` component
   - Derive `indexedFiles` and `totalChunks` from Maps using `selectedSessionId`
   - Pass `selectedSessionId` (not old sessionId) to ChatView and IndexingModal
   - When selectedSessionId is null: show empty state with "Paste a Drive link to start"
   - Key ChatView by selectedSessionId so it remounts on switch: `<ChatView key={selectedSessionId} sessionId={selectedSessionId} ...>`
  </action>
