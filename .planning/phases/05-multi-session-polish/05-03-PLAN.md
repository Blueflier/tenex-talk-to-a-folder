   - Extract drive ID with `extractDriveId(driveUrl)`
   - Fetch metadata: `GET https://www.googleapis.com/drive/v3/files/${driveId}?fields=id,mimeType` with Bearer token
   - If folder (mimeType = "application/vnd.google-apps.folder"): fetch `GET https://www.googleapis.com/drive/v3/files?q='${driveId}'+in+parents&fields=files(id)&pageSize=1000`
   - Return array of file IDs
   - On any fetch error, return empty array (graceful degradation -- indexing will catch real errors)

   - X dismiss button in top-right corner

Per CONTEXT.md: all errors use sonner toasts (no inline banners), color-coded (red=fatal, yellow=warning), auto-dismiss 5-8s for non-critical.

1. In `useStream.ts`, add 429 handling after the `response.ok` check (line ~59):
   ```
   if (response.status === 429) {
     callbacksRef.current.onError("rate_limited");
     setIsStreaming(false);
     return;
   }
   ```
   Also improve existing error handling:
   - 401/403: `callbacksRef.current.onError("auth_expired")`
   - 404: `callbacksRef.current.onError("session_not_found")`
   - Other non-ok: `callbacksRef.current.onError(\`server_error_\${response.status}\`)`
   - Catch block for network errors (not AbortError): `callbacksRef.current.onError("connection_lost")`

2. In `ChatView.tsx` (or AppShell.tsx where useStream callbacks are wired), update the `onError` callback to map error codes to appropriate toasts:
   - `"rate_limited"`: `toast.error("Too many requests. Please wait a moment.", { duration: 6000 })` + set rateLimited state
   - `"auth_expired"`: `toast.error("Session expired. Please sign in again.", { duration: 8000 })` (this may already trigger ReAuthModal)
   - `"session_not_found"`: `toast.error("Chat data not found on server. Try re-indexing your files.", { duration: 6000 })`
   - `"connection_lost"`: `toast.error("Connection lost. Check your network and try again.", { duration: 6000 })`
   - `"server_error_*"`: `toast.error("Something went wrong. Please try again.", { duration: 5000 })`

3. Add rate limit cooldown state in the component managing chat input:
   - `const [rateLimited, setRateLimited] = useState(false)`
   - On "rate_limited" error: `setRateLimited(true); setTimeout(() => setRateLimited(false), 10_000)`
   - Pass `disabled={rateLimited}` and `disabledTooltip="Rate limited -- please wait"` to ChatInput

4. In AppShell.tsx `handleIndexError`, classify indexing errors for proper toast type:
   - Errors containing "403" or "access": `toast.error("Access denied. Check file sharing permissions.", { duration: 6000 })`
   - Errors containing "404": `toast.error("File or folder not found. It may have been moved or deleted.", { duration: 6000 })`
   - Errors containing "empty": `toast.warning("This folder is empty.", { duration: 5000 })`
   - Errors containing "scanned": `toast.warning("Scanned PDFs without text can't be indexed.", { duration: 5000 })`
   - Default: `toast.error(message, { duration: 5000 })`
